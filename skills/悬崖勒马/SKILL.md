# 悬崖勒马

> 检测并打断无效循环，强制重新审视问题，避免一条道跑到黑。

## 触发规则

出现以下任一信号时，**必须立即停止当前路径**，不得继续尝试：

| # | 信号 | 判定标准 |
|---|------|---------|
| 1 | 同一操作反复失败 | 同一个工具/操作连续失败 **3 次** |
| 2 | 同类尝试兜圈子 | 用不同参数但同一思路尝试 **5 次**仍无效 |
| 3 | 报错信息重复 | 相同错误信息出现 **3 次**，说明根本问题未被解决 |
| 4 | 过度消耗 | 当前任务已超过 **15 轮**对话仍无明显进展 |
| 5 | 方向偏离 | 当前操作与用户原始需求**已无关联**（跑偏） |

## 触发后必须执行

**Step 1 — 立即停止**：不再以当前思路做任何新尝试。

**Step 2 — 回溯诊断**：用以下格式如实记录：

```
## 问题回溯

原始需求: [用户最初要我做什么]
当前卡在: [具体在哪一步卡住了]
已尝试方案:
  1. [方案A] → 失败原因: [...]
  2. [方案B] → 失败原因: [...]
  3. [方案C] → 失败原因: [...]
根本原因: [为什么前面这些方案都行不通]
```

**Step 3 — 换个方向**：至少想一个与前三次**完全不同的**解决路径。例如：
- 之前用工具A → 现在用工具B
- 之前自动化 → 现在让用户手动完成这一步
- 之前硬解 → 现在绕过这个步骤
- 之前顺序执行 → 现在并行

**Step 4 — 限制重试**：新方案只允许试 **2 次**。2 次后仍不通，必须对用户说实话：

> "我在 [某步骤] 卡住了，尝试了 [方案] 都不行，原因是 [根因]。你看我是换个思路还是你手动处理这一步？"

## 防止重复触发

如果回溯后又回到了之前试过的思路，立刻终止并告知用户。

## 使用方式

在 Hermes 对话中说：

- "开启悬崖勒马" → 加载本技能，当前对话生效
- 或者在需要长期约束时，写入 profile 的默认 skills 列表
